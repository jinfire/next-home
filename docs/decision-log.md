# Decision Log

## 2026-07-21 — 데이터 기준은 실제 수집 월을 그대로 밝힌다

연도 급지가 계산됐더라도 12개월 전체가 수집됐다고 가정하지 않는다. `trade.contract_date`에 실제 존재하는 월을 API로 제공하고 지도에 표시해 일부 월 자료를 연간 전체 자료로 오해하지 않게 한다.

## 2026-07-21 — 사용자 화면에는 도로명주소만 표시한다

국토부 원문의 법정동·지번은 내부 식별과 Geocoding 요청에 유지하되 사용자에게 표시하지 않는다. NAVER가 확인한 도로명주소를 별도 저장하고, 아직 확인되지 않았으면 지번으로 대체하지 않고 주소 표시를 생략한다.

## 2026-07-21 — 지도 지역 선택은 경계 클릭만 사용한다

지도 경계 클릭과 상단 연동 드롭다운이 같은 기능을 중복 제공하므로 지도용 시·도·시·군·구 입력은 제거한다. 현재 거주지를 정확히 지정해야 하는 갈아타기 비교의 드롭다운은 유지한다.

## 2026-07-21 — 상급지는 실제 지역과 면적별 환산 가격으로 보여준다

목표 급지의 전체 평균만 보여주지 않고 선택 지역에서 가까운 상급지 시·군·구를 최대 4곳 제공한다. 평단가 외에 25평·34평 단순 환산 가격을 함께 표시하되 특정 아파트나 매물의 실제 가격으로 표현하지 않는다.

## 2026-07-21 — 아파트 한 단계 위는 가장 가까운 상위 가격대로 정의한다

동일 시·군·구에서 현재 단지보다 평단가가 높은 후보를 오름차순으로 정렬한다. 가장 비싼 단지부터 추천하지 않고 현재 가격 바로 위 후보부터 최대 5곳을 제공한다.

## 2026-07-21 — 지도 상세에서 실거래 건수는 표시하지 않는다

`tradeCount`는 해당 기준 연도의 취소되지 않은 실거래 건수이며 급지 계산 검증을 위해 API에는 유지한다. 사용자 화면의 선택 지역 정보는 지역명·급지·평균 평단가에 집중하도록 실거래 건수 표시는 제거한다.

## 2026-07-21 — 동 급지는 원천 데이터 없이 추정하지 않는다

현재 적재된 VWorld 경계와 `region_grade`는 시·군·구 단위이므로 동 급지를 이름이나 일부 거래만으로 임의 계산하지 않는다. 우선 지도 클릭 상세와 가까운 상급지 시·군·구 4곳을 제공하고, 동 경계·법정동 매핑·충분한 연간 실거래를 적재한 뒤 동 단위 급지를 별도 구현한다.

## 2026-07-21 — 아파트 검색은 단어 순서와 일반 명칭에 관대하게 처리한다

사용자는 공식 단지명을 정확히 기억하지 않아도 된다. 검색어에서 `아파트`를 제거하고 남은 핵심 단어가 이름 또는 주소에 모두 포함되면 결과로 제공한다. 따라서 `동탄 포레나`와 `포레나 동탄`의 순서 차이로 결과가 사라지지 않는다.

## 2026-07-21 — 알림 설정은 메인 화면에서 숨긴다

서버 알림·Web Push 구현은 삭제하지 않지만 현재 사용자 흐름에서는 알림 패널을 렌더링하지 않는다. 지도, 갈아타기 비교, 같은 생활권 추천에 화면 집중도를 둔다.

## 2026-07-21 — 연도 선택은 실제 데이터가 있는 연도만 제공한다

연속된 가상 연도를 표시하지 않고 `/api/grades/years`의 결과를 오름차순 슬라이더 눈금으로 사용한다. 과거 데이터가 추가 적재되면 프런트엔드 변경 없이 자동으로 선택 범위가 늘어난다.

## 2026-07-21 — 지역과 갈아타기 입력은 연동 드롭다운을 사용한다

지역명 자유 검색과 사용자의 급지 직접 입력을 제거한다. `/api/regions/options`에서 서울·경기·인천과 하위 시군구를 받아 두 단계 드롭다운으로 선택하며, 선택한 `regionId`를 기준으로 서버가 해당 연도의 급지를 결정한다.

## 2026-07-21 — 무거래 지역도 지도에서 숨기지 않는다

수도권 전체 범위를 보여주기 위해 급지가 없는 지역도 행정구역 경계 API에 포함한다. 해당 연도의 실거래가 없으면 `grade=null`로 반환하고 지도에서는 데이터 없음 색상으로 구분한다.

## 2026-07-21 — 급지지도 범위는 수도권 전체 시군구로 한다

서비스 범위는 서울특별시·경기도·인천광역시의 전체 시군구다. VWorld의 `sig_cd` 앞 두 자리로 수도권을 선별하고 `full_nm`에서 광역단체와 시군구 표시명을 분리한다. 급지 산정과 지도는 이 83개 지역을 동일한 수도권 모집단으로 사용한다.

## 2026-07-20 — 알림 평가는 매시간, 동일 조건 발송은 24시간에 한 번으로 제한한다

- 반복 알림으로 사용자가 알림 권한을 끄는 문제를 줄인다.
- 스케줄 간격을 환경설정으로 변경할 수 있어 운영 단계에서 조정 가능하다.

## 2026-07-20 — Web Push는 자체 VAPID 키와 브라우저 Service Worker를 사용한다

### 결정

별도 유료 메시지 서비스 없이 표준 Web Push를 사용한다. 공개키는 구독 생성용 API로 제공하고 개인키는 서버 `.env`에만 둔다. 구독 endpoint는 upsert하여 중복 레코드를 만들지 않는다.

### 이유

- 브라우저가 닫혀 있을 때도 알림을 받을 수 있는 표준 방식이다.
- 외부 알림 플랫폼 비용과 계정 의존성을 추가하지 않는다.
- 개인키와 브라우저 구독 암호키의 노출 범위를 명확히 분리할 수 있다.

## 2026-07-20 — 생활권 MVP는 동일 행정구역으로 정의한다

### 결정

같은 생활권 내 상급 아파트 추천의 첫 기준은 동일한 `region_id`다. 선택 연도의 취소되지 않은 실거래 평균 평단가가 현재 아파트보다 높은 단지를 평균 평단가 내림차순으로 최대 10개 제공한다.

- 향후 교통 시간, 학군, 사용자 지정 생활권 데이터가 준비되면 기준을 확장할 수 있다.

## 2026-07-20 — 수집 거래는 source key로 중복을 방지한다

### 결정

지역코드, 아파트 식별값, 계약일, 면적, 층, 가격을 조합한 source key를 거래의 수집 고유키로 사용한다. DB 중복뿐 아니라 같은 API 페이지 안의 중복도 저장 전에 제거한다.

### 이유

- 같은 지역과 계약월을 다시 수집해도 거래가 중복 생성되지 않는다.
- 외부 API가 동일 항목을 반복 반환해도 한 번만 저장할 수 있다.

## 2026-07-20 — 실거래 금액은 원 단위 정수로 저장한다

### 결정

국토교통부 API의 만원 단위 문자열에서 쉼표를 제거하고 10,000을 곱해 원 단위 `BIGINT`로 정규화한다.

### 이유

- 가격 계산에서 부동소수점 오차를 방지한다.
- API 표시 형식과 내부 계산 단위를 분리한다.

## 2026-07-20 — DB 스키마는 Flyway와 실제 PostGIS 통합 테스트로 관리한다

### 결정

- Backend가 Flyway 마이그레이션을 실행한다.
- Collector는 스키마를 변경하지 않고 기존 스키마만 사용한다.
- 지역 경계는 geometry, 아파트 위치는 geography 타입으로 저장한다.
- 마이그레이션은 PostGIS Testcontainers 통합 테스트로 검증한다.

### 이유

- 애플리케이션과 DB 스키마 버전을 함께 관리할 수 있다.
- H2로 검증할 수 없는 공간 타입과 인덱스를 실제 PostGIS에서 확인할 수 있다.
- 두 애플리케이션이 동시에 스키마를 변경하는 충돌을 방지한다.

## 2026-07-20 — 외부 API는 콘솔과 코드에서 이중으로 차단한다

### 결정

- 네이버 콘솔의 일·월 한도를 비용 발생을 막는 최종 차단선으로 사용한다.
- Geocoding은 백엔드 호출 예산과 중복 제거를 적용하고 예산 소진 시 호출을 중단한다.
- 프론트엔드 브라우저 제한은 보조 수단으로만 사용한다.
- 한도 상향보다 캐시와 호출 절감을 먼저 검토한다.

### 이유

- 코드 오류나 키 오용이 발생해도 콘솔 한도를 넘지 않게 한다.
- 브라우저 저장소 기반 제한은 초기화하거나 우회할 수 있어 비용을 보장하지 못한다.

## 2026-07-20 — 네이버 Maps는 무료 한도 안에서만 사용한다

### 결정

- Dynamic Map: 일 10,000건, 월 100,000건
- Geocoding: 일 1,000건, 월 20,000건
- 사용량 70%부터 이메일 알림
- 사용하지 않는 Maps API는 비활성화
- 사용자 승인 없이 유료 구간으로 한도를 높이지 않음

### 이유

- 개발 단계에서 예상하지 못한 API 요금 발생을 방지한다.
- Geocoding 결과를 PostGIS에 저장하면 낮은 호출 한도로도 운영할 수 있다.

## 2026-07-20 — 급지 지도를 메인 화면으로 제공한다

### 결정

서비스의 첫 화면은 지역별 급지를 색상으로 보여주는 지도다. 지역을 선택하면 급지와 평균 평단가를 함께 보여준다.

### 이유

- 서비스의 핵심인 지역별 가격 수준을 가장 직관적으로 전달할 수 있다.
- 검색이나 복잡한 조건 입력 없이 전체 시장을 탐색할 수 있다.

## 2026-07-20 — 갈아타기 추천은 현재 급지만 입력받는다

### 결정

현재 급지를 기준으로 1급지 위와 2급지 위를 추천하고, 각 급지의 평균 평단가와 현재·과거 대비 격차를 보여준다.

개인별 대출, 보유 자금, 필요한 추가 자금은 계산하지 않는다.

### 이유

- 대출과 자금 상황은 사용자마다 달라 일관된 추천 기준으로 사용하기 어렵다.
- 객관적인 급지와 가격 격차를 제공하고 최종 판단은 사용자에게 맡긴다.
