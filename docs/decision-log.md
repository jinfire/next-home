## 2026-07-22 — 기본 지도는 네이버를 유지하고 한도 대응은 카카오 전환으로 준비한다

현재 사용자 경험은 네이버 Dynamic Map을 기본으로 유지한다. 자체 SVG 급지지도 시안은 도로·지명·배경 정보와 확대 경험이 부족해 채택하지 않는다. 네이버 호출 제한에 도달하거나 지도를 정상 로드할 수 없는 상황에는 카카오 지도로 전환하는 공급자 대체 구조를 후속 구현한다. 브라우저가 네이버의 정확한 누적 사용량을 신뢰성 있게 알 수 없으므로, 네이버 콘솔 한도·사용량 알림과 서버 운영 설정을 전환 기준으로 삼고 SDK가 명확한 한도 오류를 반환할 때도 대체 지도를 사용한다. 두 지도 SDK를 동시에 불러 API 사용량을 이중으로 발생시키지 않는다.

트래픽이 실제로 증가하기 전에는 자체 배경지도 인프라를 구축하거나 유료 한도를 확대하지 않는다. 트래픽·비용·사용자 경험을 측정한 뒤 네이버 유료 사용, 카카오 유지, MapLibre와 자체 타일, VWorld 등 대안을 다시 결정한다.

## 2026-07-22 — 장기 수집은 완료 체크포인트로 재개하고 일시 오류만 재시도한다

개발계정의 일 10,000회 한도보다 수도권 11년 지역·월 조합만 10,956개로 더 많으며 페이지 분할 호출도 필요하다. 전체 기간을 하루에 끝낸다고 가정하지 않는다. API `429`·`5xx`·네트워크 타임아웃은 제한된 횟수로 지수 백오프 재시도하고, 실패 월은 완료 처리하지 않는다. 다음 실행은 성공 체크포인트를 건너뛰어 이어받으며 페이지 크기는 1,000건으로 호출량을 줄인다.

## 2026-07-21 — 급지는 수도권 중간값 대비 가격지수로 계산한다

지역 수를 동일하게 나누는 순위 급지는 같은 급지 안의 실제 가격 차이를 숨길 수 있다. `지역 평균 평단가 ÷ 수도권 중간 평단가 × 100`을 가격지수로 사용하고 고정된 상대 구간으로 1~10급지를 배정한다. 지도에는 순위와 상위 백분위도 함께 제공하되 급지는 가격만 반영하며 학군·교통·환경의 절대 평가가 아님을 명시한다.

## 2026-07-21 — 지도와 갈아타기는 독립된 연도를 사용한다

지도 탐색 연도와 갈아타기 비교 연도는 사용 목적이 다르므로 하나의 슬라이더 상태를 공유하지 않는다. 각 기능에 독립 슬라이더를 제공하고, 아파트 추천은 과거 탐색보다 최신 시장 비교가 목적이므로 별도 슬라이더 없이 최신 계산 연도를 사용한다.

## 2026-07-21 — 우리집 평단가와 역사적 추가 금액은 동일 기간·지역 쌍으로 비교한다

우리집 평단가는 추천 단지와 동일한 최신 완결 월을 사용한다. 상급지의 역대 34평 추가 금액은 같은 두 지역에 연도별 급지가 모두 존재하는 해만 비교하며, 과거 연도가 충분하지 않으면 최소·최대라고 단정하지 않는다.

## 2026-07-21 — 최신 월은 거래 존재가 아니라 전체 수집 완료로 판단한다

새 계약월의 거래가 일부 들어온 것만으로 해당 월을 최신 비교 기간으로 인정하지 않는다. 수도권 전체 시·군·구에 대해 지역·월별 API 수집이 성공했다는 완료 기록이 모두 존재할 때만 완결된 월로 판단하고, 그중 가장 최근 월을 아파트 추천에 사용한다.

## 2026-07-21 — 아파트 추천은 최신 수집 한 달만 비교한다

최근 3개월이 연속으로 모두 수집되지 않은 상태에서 비연속 월을 합치거나 3개월 값처럼 표현하지 않는다. 선택 연도의 가장 최근 수집 월 한 달만 사용해 현재 단지와 추천 단지의 평균 평단가·거래 건수를 동일한 기간으로 비교한다.

최신 월은 2026년 6월로 고정하지 않고 DB의 최대 계약월을 매번 조회한다. 이후 최신 3개월 데이터가 연속으로 모두 수집되면 현재 단지와 추천 단지 모두 동일한 최근 3개월 기준으로 전환한다.

## 2026-07-21 — 면적별 금액은 총가격이 아니라 추가 금액이다

갈아타기 판단에 필요한 값은 상급지의 단순 총가격보다 현재 지역에서 이동할 때 벌어지는 차이다. 25평·34평 값은 두 지역의 평균 평단가 차이에 면적을 곱하고 `추가 금액`이라고 명시한다.

현재 단지보다 평당 몇 만원 높은 후보는 사용자가 기대하는 상급 단지로 보기 어렵다. 같은 지역의 상위 후보를 평균 평단가 내림차순으로 정렬해 최대 10곳 보여준다.

`래미안`, `자이`, `힐스테이트`처럼 결과가 많은 브랜드명을 전국 수도권 범위에서 바로 검색하면 최대 20개 제한 때문에 원하는 단지가 누락될 수 있다. 아파트 검색에서는 시·도와 시·군·구를 먼저 선택하고 해당 지역 안에서 이름을 부분검색한다.

## 2026-07-20 — 실거래 해제일은 실제 응답의 두 형식을 지원한다

국토교통부 실제 응답에서 해제일이 `26.01.29`처럼 `yy.MM.dd`로 내려오는 사례를 확인했다. 기존 ISO `yyyy-MM-dd` fixture 호환성을 유지하면서 두 형식을 모두 파싱한다. 실호출 검증은 페이지 크기를 전체 건수보다 크게 두어 무료 호출을 최소화한다.

## 2026-07-20 — E2E는 외부 API를 호출하지 않는다

핵심 사용자 흐름은 Playwright Chromium으로 검증하되 `/api/**` 응답을 브라우저 라우팅에서 고정한다. 테스트 실행이 NAVER Maps나 공공데이터 무료 호출량을 소비하거나 외부 장애로 불안정해지지 않도록 한다.

## 2026-07-20 — 지역 검색은 현재 연도 급지 선택과 연결한다

검색 결과 자체와 지도 급지는 별도 API로 유지한다. 사용자가 지역을 선택하면 현재 로드된 연도별 급지에서 `regionId`로 결합하며, 급지가 없는 경우 존재하지 않는 값을 추정하지 않고 안내한다.

## 2026-07-20 — 행정구역 경계는 서버에서 GeoJSON으로 제공한다

브라우저가 별도 공간 API를 호출하거나 키를 노출하지 않도록 PostGIS 경계를 Spring Boot가 GeoJSON으로 제공한다. 원천 데이터는 국토교통부 VWorld의 무료 시군구 WFS `lt_c_adsigg_info`를 서버에서 한 번 적재한다.

## 2026-07-20 — 급지 원천값은 해제 거래를 제외한 연평균 평단가로 계산한다

### 결정

### 이유

- 실제로 유지된 계약만 가격 수준에 반영한다.
- 사용자가 이해할 수 있는 단일 객관 지표를 유지한다.

## 2026-07-20 — 수집 거래는 source key로 중복을 방지한다

### 이유

- 가격이 낮은 이유가 저평가인지 실제 열위인지 객관적으로 구분하기 어렵다.
- 우선 가격 격차와 과거 데이터를 보여주고 판단은 사용자에게 맡긴다.
