# Features

## 추가 금액·추천 우선순위·실거래 월 표기 보정

- 가까운 상급지의 25평·34평 값은 상급지 총가격이 아니라 `(상급지 평단가 - 현재 지역 평단가) × 면적`으로 계산한 추가 금액이다.
- 화면 라벨을 `25평 기준 추가 금액`, `34평 기준 추가 금액`으로 명시한다.
- 아파트 추천은 몇 만원 차이의 바로 다음 단지보다 가격 수준이 확실히 높은 후보를 우선하도록 평단가 내림차순으로 되돌리고 최대 10곳을 제공한다.
- 수집 월이 1월과 6월처럼 떨어져 있으면 `1월·6월 실거래 합계`, 1~6월처럼 연속이면 `1~6월 실거래 합계`로 구분한다.

## 지역을 먼저 선택하는 아파트 검색

- `래미안`처럼 같은 이름이 많은 브랜드 단지를 정확히 찾기 위해 아파트 검색 전에 시·도와 시·군·구를 선택한다.
- 선택한 `regionId`를 `/api/apartments`에 전달해 해당 시·군·구 안에서만 이름 부분검색을 수행한다.
- 지역을 선택하기 전에는 아파트명 입력과 검색 버튼을 비활성화한다.
- 지역을 변경하면 이전 검색 결과·현재 단지·추천 결과를 초기화해 다른 지역 결과가 섞이지 않게 한다.
- 서버는 최대 20개를 반환하지만 지역 필터를 먼저 적용하므로 흔한 브랜드명의 불필요한 결과를 줄인다.

## 아파트 결과 표시 보정

- 아파트 검색 API 응답에서 원본 지번 `address` 필드를 제거해 프런트엔드가 실수로 지번주소를 다시 표시할 수 없게 한다.
- 검색 입력 예시는 실제 결과가 존재하는 `포레나`로 제공한다.
- 추천 단지의 거래 수에는 실제 수집 월을 붙여 `6월 실거래 5건` 또는 `1·6월 실거래 5건`처럼 표시한다.
- 최신 갈아타기 API에서 2급지 지역은 가까운 1급지 후보를 최대 4곳 반환하며 실제 실행 API로 확인한다.

## 실제 수집 월과 아파트 도로명주소

- 급지 지도는 선택 연도의 전체 자료가 있는 것처럼 표현하지 않고 실제 DB에 존재하는 계약월을 `2026년 1·6월 실거래 기준`처럼 표시한다.
- 아파트의 기존 `address`는 국토부 실거래 원문의 법정동·지번으로 내부 식별에 유지하고 사용자 화면에는 노출하지 않는다.
- NAVER Geocoding 결과의 `roadAddress`를 아파트에 별도 저장해 검색 결과·선택 단지·추천 단지 아래에 표시한다.
- 도로명주소가 아직 확인되지 않은 단지는 지번으로 대체하지 않고 주소 줄을 숨긴다.
- 갈아타기 화면은 구버전 백엔드 응답에 `nearbyRegions`가 없어도 화면 전체가 중단되지 않는다.

## 지도 지역 드롭다운 제거

- 지도는 행정구역 경계를 직접 클릭해 지역을 선택하므로 상단의 `지도 지역 시·도 / 시·군·구` 드롭다운을 표시하지 않는다.
- 갈아타기 비교에서는 현재 거주 지역을 명확하게 지정해야 하므로 시·도→시·군·구 드롭다운을 유지한다.

## 구체적인 상급지·아파트 한 단계 추천

- 갈아타기 화면은 추상적인 목표 급지 평균 카드 대신 선택 지역에서 경계 중심점 기준으로 가까운 상급지 시·군·구를 최대 4곳 표시한다.
- 각 상급지에는 지역명·급지·평단가와 함께 `평단가 × 25`, `평단가 × 34`로 계산한 25평·34평 기준 가격을 표시한다. 이는 지역 평균의 단순 환산값이며 특정 매물 가격이 아니다.
- 아파트 추천은 현재 단지보다 비싼 후보를 평단가 내림차순으로 정렬해 가격 수준이 확실히 높은 단지부터 최대 10곳을 보여준다.
- 아파트 검색 DB는 실거래 수집에서 발견한 단지로 구성되므로 수집 기간에 거래가 없던 단지는 검색되지 않을 수 있다. 정확하지 않은 유사 단지를 대신 반환하지 않는다.

## 지도 클릭과 아파트 부분검색

- 연도 슬라이더는 2015년부터 현재 연도까지 항상 이동할 수 있고, 데이터가 없는 연도는 추정하지 않고 빈 급지 상태로 표시한다.
- 수도권 지도 경계를 클릭하면 같은 화면 오른쪽 정보 패널에서 지역명·급지·평균 평단가를 확인한다. 연간 실거래 수는 계산과 API에는 유지하지만 화면에는 표시하지 않는다.
- 갈아타기 결과는 선택 지역 경계 중심점에서 가까운 상급지 시·군·구를 최대 4곳만 보여준다.
- 아파트 검색은 입력 문장을 공백 단위 핵심어로 나누고 순서와 무관하게 이름·저장 주소에 모든 핵심어가 포함된 단지를 찾는다.
- 사용자가 `동탄 포레나 아파트`처럼 입력하면 일반 명칭 `아파트`를 제외하고 `동탄`, `포레나`가 포함된 실제 단지 목록에서 선택할 수 있다.
- 현재 경계와 급지 모집단은 시·군·구 단위다. 읍·면·동 급지는 동 경계와 동별 가격 집계가 추가된 뒤 제공한다.

## 수도권 급지지도 사용자 화면

- 첫 화면 제목은 `수도권 급지지도`이며 서울·경기·인천 전체 경계를 한 지도에서 보여준다.
- 지도와 갈아타기 비교는 자유 텍스트 대신 `시·도 → 시·군·구` 연동 드롭다운을 사용한다.
- 지도 연도 슬라이더는 `/api/grades/years`가 반환한 실제 보유 연도만 선택할 수 있다. 한 연도만 있으면 값은 표시하되 조작은 비활성화한다.
- 선택 지역의 급지·평균 평단가·실거래 수를 지도 아래에 요약하고 1~10급지 및 데이터 없음 색상을 범례로 제공한다.
- 갈아타기 비교는 선택한 지역의 급지를 서버가 자동 계산하며 1·2급지 위 평균 평단가, 현재 격차, 과거 격차 수준을 표시한다.
- 알림 기능 코드는 보존하지만 사용자가 요청한 현재 제품 범위에서는 알림 설정 화면을 렌더링하지 않는다.
- 같은 생활권 추천 검색은 저장된 주소 또는 아파트명으로 조회한다.

## 수도권 실거래와 자동 급지 비교

- `collectCapitalAreaTrades` 명령이 `.env`의 시작·종료 월을 기준으로 수도권 83개 지역을 순차 수집한다.
- source key 중복 검사를 유지하므로 중단 뒤 같은 기간을 재실행해도 기존 거래는 건너뛴다.
- 실제 2026년 6월 수집에서 거래 23,032건을 신규 저장하고 중복 1,178건을 제외했다.
- `recalculateGrades` 명령은 거래가 있는 모든 연도의 수도권 급지를 다시 계산한다.
- 2026년은 82개 지역에 급지가 계산됐고 거래가 없는 옹진군도 지도에서 회색 경계로 표시된다.
- `/api/regions/options`가 서울·경기·인천과 하위 시군구를 중첩 목록으로 제공한다.
- 갈아타기 API는 `regionId`로 현재 지역의 급지를 자동 조회해 1·2급지 위 격차를 반환한다.
- 아파트 검색은 단지명과 저장 주소를 함께 검색한다.

## 수도권 전체 시군구 경계

- VWorld 시군구 WFS 한 번의 요청에서 서울·경기·인천 지역만 선별한다.
- 서울특별시·경기도·인천광역시를 1단계 지역으로 만들고 시군구를 2단계 자식 지역으로 저장한다.
- `성남시 분당구`처럼 시 이름이 필요한 일반구는 전체 지역명을 보존하고, 동명 지역은 상위 시도로 구분한다.
- 실제 로컬 PostGIS에는 서울 25개, 경기 47개, 인천 11개로 총 83개 경계가 적재됐다.
- 재실행 시 새 행을 중복 생성하지 않고 지역명·부모·경계를 갱신한다.

## 실제 로컬 스택 브라우저 검증

- 선택 실행되는 `live-stack.spec.ts`가 mock 없이 로컬 PostgreSQL·Spring Boot·React·NAVER Map을 함께 검증한다.
- 실제 DB의 수도권 경계 83개와 2026년 급지, 시·도→시·군·구 선택 목록이 브라우저에 표시됨을 확인했다.
- 기본 E2E에는 포함하지 않아 외부 지도 호출이 자동으로 반복되지 않는다.

## VWorld 시군구 경계 수집기 구현

- 공식 무료 WFS 레이어 `lt_c_adsigg_info`를 대한민국 범위, EPSG:4326, GeoJSON 형식으로 요청한다.
- GeoJSON의 `sig_cd`를 기존 `region.code`와 매칭하고 Polygon을 PostGIS MultiPolygon으로 정규화해 저장한다.
- 매칭되지 않는 코드는 건너뛰며, 전용 `importVworldBoundaries` 명령에서만 외부 요청이 발생한다. 일반 서버 시작은 VWorld를 호출하지 않는다.
- 원천: [공공데이터포털 국토교통부 행정구역도](https://www.data.go.kr/data/15059008/openapi.do)
- 실제 WFS 1회 호출로 현재 DB의 종로구 경계를 적재했으며, `ST_MultiPolygon`·SRID 4326·유효 도형·약 23.99㎢를 확인했다.
- 공식 WFS 1.1.0 명세에 따라 GeoJSON은 `output=application/json`, EPSG:4326 bbox는 `33,124,39,132` 순서로 요청한다.

## 국토교통부 실거래 실제 수집 검증

- 공식 상세 기능 경로 `/getRTMSDataSvcAptTradeDev`에서 인증 성공 코드 `000`을 확인했다.
- 종로구(11110) 2026년 1월 응답 46건을 한 페이지로 받아 중복 1건을 제외하고 거래 45건과 아파트 26개를 로컬 PostgreSQL에 저장했다.
- 실제 응답의 해제일 `yy.MM.dd`와 기존 ISO 날짜를 모두 해석하며, 해당 회귀 테스트를 추가했다.

## 핵심 사용자 흐름 E2E 검증

- 실제 Chromium에서 메인 급지 표시, 지역 검색·선택, 현재 급지 입력, 1급지 위 가격 격차 표시를 한 흐름으로 검증한다.
- E2E에서는 네트워크 라우팅으로 API 응답을 고정하여 외부 지도·공공데이터 API 사용량을 소비하지 않는다.
- 브라우저 검증 과정에서 발견된 NAVER Data API 오용(`removeAll`)을 공식 메서드인 `removeFeature`로 교정했다.

## 기존 지역 검색 화면 구현(연동 드롭다운으로 대체됨)

- 기존 자유 텍스트 검색은 보존된 API 호환 기능이며 현재 화면에서는 사용하지 않는다.
- 검색 결과를 선택하면 해당 연도의 급지 목록과 연결해 선택 요약을 갱신한다.
- 선택한 지역에 해당 연도 급지가 없으면 빈 선택 대신 명확한 안내를 표시한다.

## 행정구역 급지 색상 오버레이 구현

- `GET /api/region-boundaries?year=2026`은 경계가 저장된 지역과 해당 연도 급지를 GeoJSON FeatureCollection으로 반환한다.
- 지도는 기준 연도가 바뀔 때 기존 경계를 제거하고 새 경계를 1~10급지 색상으로 표시한다.
- 경계가 없는 지역은 응답에서 제외되어 잘못된 도형이나 빈 Feature를 만들지 않는다.
- 실제 경계 원천은 국토교통부 VWorld의 무료 시군구 WFS `lt_c_adsigg_info`를 사용하며, 서버에서 1회 적재해 브라우저에 키를 노출하지 않는다.

## Dynamic Map 호출 절약 구현

- 지도 영역이 화면 200px 이내에 들어올 때만 NAVER Maps SDK를 지연 로딩한다.
- 동시에 여러 지도 소비자가 생겨도 공유 Promise와 단일 script 태그를 사용한다.
- 한 지도 컴포넌트에서는 지도 객체를 한 번만 생성한다.

## Geocoding 캐시와 무료 한도 보호 구현

- 좌표가 없는 아파트 주소를 백엔드 배치가 한 번에 최대 50건씩 NAVER Geocoding으로 변환한다.
- 공백을 정규화한 주소를 `geocoding_cache`에 저장하여 같은 주소는 외부 API를 다시 호출하지 않는다.
- 코드 한도는 콘솔 한도보다 낮은 일 900건, 월 18,000건이며 둘 중 하나라도 소진되면 외부 호출 전에 즉시 중단한다.
- 좌표는 아파트의 위도·경도와 PostGIS `geography(Point, 4326)`에 함께 저장한다.
- 브라우저는 Geocoding API나 Client Secret을 직접 사용하지 않는다.

## 백그라운드 갈아타기 알림 구현

서버가 활성 알림 조건을 기본 한 시간 간격으로 평가한다. 현재 지역과 목표 지역 또는 목표 급지의 연도별 평균 평단가로 현재 격차율과 과거 백분위를 계산하며 설정된 모든 임계값 이하일 때만 알린다. Web Push payload는 브라우저 구독키로 암호화하고 서버 VAPID 개인키로 서명한다. 같은 조건이 계속 충족되어도 24시간 이내에는 다시 발송하지 않는다.

## Web Push 구독 구현

브라우저 알림 권한을 허용하면 Service Worker를 등록하고 서버에서 공개 VAPID 키를 조회한다. PushManager 구독을 생성한 뒤 endpoint와 브라우저 공개키를 `push_subscription` 테이블에 저장한다. 동일 endpoint를 다시 등록하면 브라우저 식별자와 키를 갱신한다. VAPID 개인키는 로컬 `.env`와 서버에서만 사용하며 API 응답이나 프런트 번들에 포함하지 않는다.

## 웹 알림 1단계 구현

사용자는 현재 급지와 허용할 최대 평당 격차(만원/평)를 설정한다. 조건은 브라우저 `localStorage`에 저장하며 권한 허용 후 즉시 조건을 확인한다. 페이지가 열린 동안 한 시간마다 선택 연도의 갈아타기 추천을 다시 조회하고 목표 급지와의 평당 격차가 설정값 이하이면 Notification API로 알린다. 브라우저를 완전히 닫은 상태의 알림은 Service Worker와 서버 Web Push를 연결하는 후속 단계다.

## 같은 생활권 추천 화면 구현

아파트 이름을 검색해 현재 단지를 선택하면 선택 연도의 동일 행정구역 추천 API를 호출한다. 추천 결과에는 단지명과 주소, 평균 평단가, 현재 단지 대비 평당 격차, 유효 실거래 수를 표시한다. 유효 거래가 없거나 후보가 없는 경우도 별도 상태로 안내한다.

## 갈아타기 비교 화면 구현

현재 급지 하나를 선택하면 선택 연도의 1급지 및 2급지 위 후보를 API에서 조회한다. 결과 카드에는 목표 급지 평균 평단가, 현재 평당 격차, 과거 격차 백분위와 관측 연도 수만 표시한다. 개인별 차이가 큰 대출 가능액과 필요 추가 자금은 계산하거나 표시하지 않는다.

## 메인 급지 지도 화면 구현

- 첫 화면의 중심을 네이버 Dynamic Map과 지역 급지 목록으로 구성한다.
- 기준 연도를 바꾸면 `/api/grades`에서 해당 연도의 급지 데이터를 다시 읽는다.
- 지역별 급지, 평균 평단가와 유효 실거래 수를 함께 표시한다.
- 지역을 선택하면 현재 선택한 지역의 핵심 값을 지도 아래에 요약한다.
- 행정구역 경계는 PostGIS `boundary`와 연도별 급지를 결합해 지도 오버레이로 표시한다.

## 갈아타기 웹 알림 조건 등록

- `browserId`, 현재 지역과 목표 지역 또는 목표 급지를 저장한다.
- 목표 격차율과 과거 격차 백분위는 선택 조건으로 저장할 수 있다.
- 목표 지역 또는 목표 급지 중 하나는 반드시 입력해야 한다.
- 이 단계는 조건 등록까지 구현하며, 주기적 평가와 Web Push 발송은 후속 기능이다.

## 같은 생활권 내 상급 아파트 추천 구현 기준

- MVP의 생활권은 동일한 행정구역(`region_id`)으로 정의한다.
- 선택 연도의 취소되지 않은 실거래로 아파트별 평균 평단가를 계산한다.
- 현재 아파트보다 평균 평단가가 높은 단지만 추천한다.
- 평균 평단가가 높은 순으로 최대 10개를 제공한다.
- 추천 단지의 평균 평단가, 거래 수, 현재 단지 대비 평당 격차를 표시한다.
- “상급”은 객관적인 가격 지표만 뜻하며 학군·교통·주거 품질을 단정하지 않는다.

> 이 문서는 만들기로 결정한 기능과 이후 후보 기능을 정리한다.

## MVP

### 급지 지도

- 서비스의 메인 화면으로 제공한다.
- 지역별 급지를 지도 색상으로 표시한다.
- 연도 슬라이더로 과거 급지를 확인한다.
- 급지는 평단가만 기준으로 계산한다.
- `가격 ÷ (전용면적㎡ ÷ 3.305785)`로 거래별 평단가를 계산한다.
- 해제 거래를 제외한 지역별 연평균 평단가를 사용한다.
- 평균 평단가가 높은 지역부터 1급지, 가장 낮은 지역을 10급지로 배정한다.
- 지역을 선택하면 해당 급지와 평균 평단가를 보여준다.

### 갈아타기 추천

사용자에게 현재 지역 또는 현재 급지만 입력받는다.

- 현재보다 1급지 위와 2급지 위 지역을 나누어 보여준다.
- 현재 급지, 1급지 위, 2급지 위의 평균 평단가를 보여준다.
- 각 상급지와의 현재 평균 평단가 격차를 보여준다.
- 현재 격차가 과거와 비교해 어느 수준인지 보여준다.
- 대출, 보유 자금, 필요한 추가 자금은 계산하지 않는다.
- 과거 대비 격차는 백분위로 제공하며 `0%`는 관측 기간 중 최소, `100%`는 최대 격차를 뜻한다.

### 갈아타기 알림

- 사용자가 현재 급지, 목표 급지 또는 관심 지역을 등록한다.
- 목표 급지와의 가격 격차가 사용자가 정한 조건에 도달하면 알려준다.
- 알림 조건에는 목표 격차와 과거 대비 격차 수준을 사용할 수 있다.
- 웹 알림으로 제공한다.
- 브라우저 알림 권한을 허용한 사용자에게 조건 도달 알림을 표시한다.

### 같은 생활권 내 더 좋은 아파트 추천

- 현재 아파트와 같은 생활권 안에서 더 높은 가격 수준의 아파트를 추천한다.
- 후보 아파트의 평균 평단가와 현재 아파트 대비 가격 격차를 보여준다.
- 생활권의 정의와 아파트 비교 기준은 데이터 확보 범위에 맞춰 구체화한다.

### 격차 변화

두 지역의 가격 차이가 연도별로 어떻게 변했는지 보여준다.

예시:

- 2018년: 0.8억
- 2020년: 1.5억
- 2022년: 3억
- 2026년: 2억

### 역사적으로 지금은?

현재 가격 격차가 과거와 비교해 어느 수준인지 보여준다.

예시:

- 최근 15년 중 세 번째로 작은 격차
- 최근 10년 중 가장 큰 격차

### 갈아타기 지수

현재 갈아타기 조건을 점수로 표현한다.

초기에는 다음 요소를 활용한다.

- 현재 격차
- 과거 대비 격차 위치
- 최근 격차 추세
