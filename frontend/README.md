# Frontend

갈아타기 비교는 목표 급지 숫자만 보여주지 않고 선택 지역에서 가까운 상급지 시·군·구를 최대 4곳 보여준다. 지역 평균 평단가와 25평·34평 단순 환산 가격을 함께 확인할 수 있다.

아파트 추천은 검색 결과에서 정확한 현재 단지를 먼저 선택한 뒤 같은 시·군·구의 바로 다음 상위 평단가 단지를 최대 5곳 표시한다. 현재 검색 대상은 실거래 수집 과정에서 저장된 단지이므로 수집 기간에 거래가 없었던 단지는 나오지 않을 수 있다.

지도 경계를 클릭하면 오른쪽 패널에서 선택 지역의 급지와 평균 평단가를 확인한다. 연간 실거래 수는 화면에 노출하지 않는다. 연도 슬라이더는 2015년부터 현재까지 이동하며, 적재되지 않은 연도는 데이터를 추정하지 않는다.

아파트 추천 검색은 공식 이름 전체가 아니라 `동탄 포레나` 같은 일부 핵심어만 입력해도 된다. 결과 목록에서 실제 단지를 선택한 뒤 같은 시·군·구의 더 높은 평단가 단지를 조회한다.

## 현재 메인 화면

`수도권 급지지도`를 제목으로 서울·경기·인천 83개 시군구 경계를 한 화면에 표시한다. 지도와 갈아타기 비교의 지역 입력은 자유 검색이 아니라 시·도와 시·군·구를 차례로 고르는 연동 드롭다운이다. 연도 슬라이더는 백엔드에 급지가 계산된 실제 연도만 보여준다.

현재 화면에서는 갈아타기 알림 설정을 노출하지 않는다. 아래 알림 관련 설명은 보존된 서버·컴포넌트 구현에 대한 참고다.

## 서버 알림 조건 연결

지도에서 선택한 현재 지역과 급지, 사용자가 입력한 최대 평당 격차를 서버용 격차율로 변환한다. 1급지 위와 2급지 위 조건을 `/api/alerts`에 저장하므로 페이지나 브라우저가 닫힌 뒤에도 서버 Web Push를 받을 수 있다.

## Service Worker Push 구독

웹 알림을 켜면 `/sw.js`를 등록하고 VAPID 공개키로 PushManager 구독을 만든다. 구독은 백엔드에 저장되며 Push 수신 시 시스템 알림을 표시하고 클릭하면 Next Home 창을 연다.

## 웹 알림

현재 급지와 최대 평당 격차를 설정하면 페이지가 열린 동안 한 시간마다 조건을 확인한다. 조건 충족 시 브라우저 Notification API로 알리며 설정은 현재 장치의 `localStorage`에 저장된다. 브라우저가 닫힌 상태의 Web Push는 아직 후속 작업이다.

## 같은 생활권 아파트 추천

현재 아파트를 검색하고 선택하면 같은 행정구역의 더 높은 평균 평단가 단지를 최대 10개 표시한다. 추천의 “상급”은 가격 데이터 기준이며 생활 품질을 단정하지 않는다.

## 갈아타기 비교

현재 급지를 선택하면 `/api/recommendations/upgrades`에서 1·2급지 위 평균 평단가, 현재 격차와 과거 격차 백분위를 읽어 온다. 개인 대출 정보나 추가 자금은 입력받지 않는다.

## Next Home 급지 지도

메인 화면은 네이버 Dynamic Map과 연도별 지역 급지 목록을 보여준다. 루트 `.env`의 `NAVER_MAP_CLIENT_ID`를 사용하며 지도 Client Secret은 프런트엔드에 포함하지 않는다. 개발 서버의 `/api` 요청은 같은 `.env`의 `BACKEND_PORT`로 전달된다.

Dynamic Map SDK는 지도 영역이 화면 가까이에 들어올 때 지연 로딩하며, 페이지 안에서 script와 SDK 로드 요청을 한 번만 공유한다.

기준 연도의 행정구역 GeoJSON이 있으면 NAVER Maps Data 레이어에서 1~10급지 색상으로 표시한다. 연도 변경 시 경계도 함께 갱신된다.

지도 상단의 연동 드롭다운은 `/api/regions/options` 결과를 현재 연도 급지 목록과 연결해 선택 요약을 갱신한다.

## 브라우저 E2E 테스트

최초 한 번 `npx playwright install chromium`으로 브라우저를 설치한 뒤 실행한다.

```powershell
npm run test:e2e
```

테스트는 API 응답을 가로채므로 외부 API 호출량을 사용하지 않는다.

로컬 PostgreSQL의 실제 수집 데이터와 NAVER Map까지 확인하는 선택적 스모크 테스트는 백엔드를 루트 `.env`의 `BACKEND_PORT`에서 실행한 상태로 다음처럼 실행한다. 이 명령은 Dynamic Map을 실제 1회 로드한다.

```powershell
$env:LIVE_E2E='1'
npm run test:e2e -- live-stack.spec.ts
```

```powershell
npm run dev
npm test
npm run build
```
